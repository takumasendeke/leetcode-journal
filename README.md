# leetcode journal

just a space to track my algorithmic problem-solving. 

this isn't a database of perfect, copy-pasted solutions. it's a log of my actual thought process, focusing on big o complexity, memory management, and first-principles logic in raw c++.

### the workflow
every problem is broken down into two parts:
1. the raw `solution.cpp` file.
2. a `notes.md` file breaking down the math, the memory, and why i chose that specific approach.

### the rules
no autocomplete. no shortcuts. NO AI!. every solution is written from scratch, compiled locally, and debugged manually to build actual muscle memory.
